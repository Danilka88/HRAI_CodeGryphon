# HR Vacancy Builder

Бесплатный open-source инструмент для HR-менеджеров, который помогает быстро создавать требования к вакансиям и анализировать резюме с помощью локальной модели Ollama. Проект ориентирован на практическую работу: от формулировки вакансии до структурированного сравнения кандидата с требованиями — в одном удобном веб-интерфейсе.

## 🏆 [Видео обзор проекта](https://disk.yandex.ru/i/WT0RZbGo3lPH9w)

## 1. Заголовок и краткое описание проекта

**HR Vacancy Builder** — это локальный веб-инструмент для HR-команд и рекрутеров, который автоматизирует подготовку требований к вакансии и первичный анализ резюме.

Проект помогает:

- быстрее запускать подбор по новым позициям;
- стандартизировать требования по ролям;
- проводить первичную оценку кандидатов на основе единых критериев;
- сохранять историю работы для повторного использования лучших формулировок.

## 2. Ключевые возможности и функционал

Ниже — основные функции системы:

- 🤖 **Генерация требований к любой вакансии через локальную Ollama** (без внешних API)
- 🧩 **Редактируемые карточки требований** (подтверждение, отклонение, правка)
- 📝 **Создание готового Markdown-документа для вакансии**
- 🔍 **Анализ резюме кандидата** с выделением сильных и слабых сторон
- 🗃️ **Полная история всех вакансий и анализов в IndexedDB**
- 🏆 **Функция «Найти лучшую версию»** для вакансий с одинаковым названием/запросом
- ⚖️ **Сравнение похожих вакансий** и выбор наиболее качественной версии
- ⏳ **Прогресс-бар/индикатор выполнения при работе с LLM** (визуальный статус обработки)
- 🇷🇺 **Полностью русский интерфейс**
- 💾 **Автоматическое сохранение всех данных** в локальном хранилище браузера
- 🧪 **Импорт резюме из hh.ru с безопасным демо-режимом по умолчанию**
- 🔐 **Поддержка переключения demo/API-режима для hh.ru** (реальные запросы включаются только при наличии API-ключа)

Дополнительно:

- 📚 Архив с фильтрацией записей (вакансии/анализы)
- 📄 Экспорт результатов в `.md`
- 🔁 Повторное открытие и доработка сохранённых материалов
- 📊 CSV-экспорт анализов резюме с экранированием кавычек и защитой от формул таблиц (`=`, `+`, `-`, `@`)

## 3. Главное преимущество — приватность

> [!IMPORTANT]
>
> ### 🔐 Полная приватность данных
>
> - Это **полностью open-source проект**.
> - **Защита от вендоров и облачных сервисов**: нет привязки к внешним API-провайдерам.

Это особенно важно для HR-процессов, где резюме и внутренние требования к вакансии содержат чувствительную информацию.

## 4. Как запустить проект

Ниже — пошаговая инструкция для быстрого старта.

### Шаг 1. Установите Ollama

1. Перейдите на официальный сайт Ollama: `https://ollama.com/`.
2. Скачайте и установите версию для вашей ОС.
3. Убедитесь, что сервис Ollama запущен.

Проверка в терминале:

```bash
ollama --version
```

### Шаг 2. Скачайте модели

Проект работает с локальными моделями Ollama. Примеры поддерживаемых моделей:

```bash
ollama pull qwen3.5:0.8b
ollama pull gemma3:4b
ollama pull qwen3.5:4b
```

При необходимости вы можете использовать другие модели, совместимые с Ollama.

### Шаг 3. Скачайте/откройте проект

Если проект уже находится локально, перейдите в папку проекта:

```bash
cd hr-vacancy-builder
```

### Шаг 4. Запустите локальный веб-сервер

Из корня проекта запустите:

```bash
python -m http.server 8000
```

### Шаг 5. Откройте приложение в браузере

Откройте страницу:

```text
http://localhost:8000/hr-vacancy-builder/
```

Если используется другой путь/порт — скорректируйте URL под вашу структуру папок.

### Шаг 6. Проверьте подключение к Ollama

Приложение обращается к локальному Ollama по адресам:

- `http://localhost:11434`
- `http://127.0.0.1:11434` (fallback)

Если генерация не запускается:

- проверьте, что Ollama запущен;
- проверьте, что нужная модель загружена (`ollama list`);
- проверьте, что порт `11434` доступен локально.

### Шаг 7. Используйте импорт резюме из hh.ru (demo/API)

В интерфейсе анализа резюме доступен отдельный блок **«Импорт резюме из hh.ru»**:

1. По умолчанию включён **демо-режим** (`Использовать демо-данные`), где можно протестировать сценарий без внешних запросов.
2. Для реального API-режима отключите демо-режим и укажите **API-ключ hh.ru**.
3. Заполните параметры поиска: запрос (`query`), регион (`area`), количество (`per_page`).
4. Нажмите «Загрузить резюме», затем выберите запись и примените её в поле резюме.

> [!NOTE]
> API-ключ используется только для обращений к hh.ru в API-режиме и не должен публиковаться в открытом доступе.

## 5. Требования к системе

Минимальные требования:

- **ОС:** Windows, macOS или Linux
- **Браузер:** современный браузер с поддержкой ES Modules (Chrome, Edge, Firefox, Safari)
- **Python:** 3.x (для запуска `python -m http.server`)
- **Ollama:** установлен и запущен локально
- **Модель Ollama:** минимум одна загруженная модель (например, `qwen3.5:0.8b`)
- **Сеть:** интернет может понадобиться только для первоначальной установки Ollama/моделей; рабочий режим — локальный

Рекомендуется:

- 8+ ГБ RAM для комфортной работы с более крупными моделями
- SSD и актуальная версия браузера для лучшей отзывчивости интерфейса
- **Node.js 20+** для локальной проверки синтаксиса JavaScript-модулей (`node --check`)

## 7. Проверки и диагностика (актуализировано)

### 7.1 Быстрая проверка доступности Node.js

```bash
node --version
```

Если команда не найдена (`command not found: node`), это проблема окружения/`PATH`, а не ошибка кода приложения.

### 7.2 Проверка синтаксиса всех JS-модулей

Из корня проекта:

```bash
for file in hr-vacancy-builder/js/*.js; do
  node --check "$file"
done
```

Признак успешной проверки: все файлы проходят без ошибок синтаксиса.

### 7.3 Fallback-проверка без Node.js

Если Node.js недоступен, выполните структурную проверку через проектные инструменты:

- проверить наличие ключевых UI-элементов hh.ru в `index.html`;
- проверить импорты/экспорты между `main.js`, `state.js`, `resume-analysis.js`, `ui-renderer.js`, `hh-resume-provider.js`;
- проверить присутствие ключевых функций (`requestResumesFromHh`, `formatHhResumeForTextarea`, `fetchHhResumes`, `applySelectedHhResume`).

Этот режим позволяет подтвердить целостность интеграции до установки Node.js.

## 8. Безопасность и практические рекомендации

- Не храните и не публикуйте API-ключ hh.ru в репозитории или публичных документах.
- Используйте демо-режим для презентаций и тестов без реальных персональных данных.
- При экспорте CSV учитывайте защиту от формульных инъекций: значения, начинающиеся на `=`, `+`, `-`, `@`, должны быть префиксованы.
- Для локальной работы с чувствительными резюме используйте отдельный профиль браузера и ограниченный доступ к рабочей машине.
